# 流放之路2 剧情流程辅助 — 页面交互方案

| 关联文档 | PRD v0.2 |
|----------|----------|
| 更新日期 | 2026-05-25 |
| 范围 | 主工作台（章节 + 地图 + 详情），不含代码实现 |

---

## 一、核心交互原则（本次修订）

1. **固定主线 Flow**：每章有一条预先排好的节点顺序（①→②→③…），全站一致，用户按此顺序推进。
2. **用户声明当前位置**：可随时点击地图/列表中的节点，视为「我认为自己在这里」；必要时一键「设为我的当前节点」对齐进度指针。
3. **游戏过一个节点 → 网站点一次「下一步」**：主循环是**节点级**前进，不是逐步解锁子步骤。
4. **「上一步」用于查漏补缺**：沿固定 Flow 回退查看上一节点详情，不强制打断查阅。
5. **右侧详情必须支持配图**：节点说明、步骤说明均可带图，可点击放大。
6. **中间地图可缩放**：游戏截图支持放大/缩小/适应/复位，编号点随图锚定。

## 二、页面整体布局（桌面端）

```
┌──────────────┬────────────────────────────┬──────────────────────────┐
│  左：章节导航  │  中：地图（截图 + 编号）       │  右：节点详情 + 配图 + 操作  │
│  Act 1 ▼     │  [－] 100% [＋] 适应 复位     │  ▶ 下一步提示（当前节点）   │
│  Flow 列表    │  ┌────────────────────┐   │  [头图 / 配图轮播]         │
│  ①②③…同序   │  │  游戏地图截图        │   │  标题 · 标签              │
│              │  │   ①  ②  ③          │   │  文字说明 · 步骤列表        │
│              │  └────────────────────┘   │  [上一步]  [下一步]        │
└──────────────┴────────────────────────────┴──────────────────────────┘
```

| 区域 | 职责 |
|------|------|
| **左** | Act 切换；与 Flow **同序**的节点列表（序号与地图编号一致） |
| **中** | 空间对照：截图、编号、缩放/平移 |
| **右** | 怎么做：文字 + **图片** + 沿 Flow 前进/回退 |

**边玩边查视线**：右栏「下一步提示」→ 需要认路时看中栏放大图 → 换章/总览看左栏。

## 三、固定 Flow 与进度标记

### 3.1 固定 Flow 是什么

- 每个 Act 有一条**只读**的主线列表，例如：`[节点A, 节点B, 节点C, …]`。
- 地图上的 **① ② ③** 与左侧列表序号，均对应该 Flow 中的顺序（同一节点在全站共用一个序号）。
- 用户**不能**通过随意跳关改变 Flow 顺序，只能通过「下一步 / 上一步」或「设为当前」移动**进度指针**。

### 3.2 两类「当前」概念（用户需能区分）

| 概念 | 用户理解 | 界面表现 |
|------|----------|----------|
| **进度指针** | 我剧情推到 Flow 哪一节 | 列表/地图上「当前进度」脉冲；「下一步」从这里往后走 |
| **正在查看** | 我正在看哪节的攻略 | 右栏显示该节详情与配图；可能与指针不同（查阅模式） |

### 3.3 状态在界面上的标记

| 状态 | 地图编号 | 左侧列表 | 右栏 |
|------|----------|----------|------|
| **已完成** | 勾 + 弱化 | 勾 + 可选划线 | 可点开复查 |
| **当前进度** | 脉冲高亮 | 「当前」徽标 | 默认「下一步提示」指向此节点 |
| **未到** | 实心灰或空心 | 正常字重 | 可查看（剧透关时可能禁止） |
| **可选支线** | 「支」角标 | 分组在 Flow 下方 | 不影响主线指针 |

## 四、用户操作流程

### 流程 A：首次打开

1. 用户打开网站 → 首页「开始剧情」或「继续」。
2. 进入 **Act 1** 工作台；进度指针在 Flow **第 1 节**；正在查看 = 第 1 节。
3. 中栏加载第 1 节所在**地图截图**，编号 **①** 高亮为当前进度。
4. 右栏展示第 1 节：**下一步提示** + 头图/配图 + 文字步骤说明。
5. 用户无需点击即可对照游玩。

---

### 流程 B：边玩边查主循环（下一步 · 节点级）

1. 用户看右栏 **「下一步提示」**（当前节点要做什么）及配图。
2. 需要认路时，在中栏 **放大地图**（见流程 L），对照编号位置。
3. 用户在**游戏内完成该节点**（打完、交任务、开传送点等）。
4. 用户回到网站，点击右栏 **「下一步」**。
5. 系统行为（用户可感知）：
   - 刚完成的节点标记为 **已完成**（打勾）；
   - 进度指针移到 Flow **下一节**；
   - 右栏自动切换为下一节详情与配图；
   - 「下一步提示」更新为下一节文案；
   - 中栏若下一节在不同截图，**自动切换地图**并高亮下一编号（如 ②→③）；
   - 左侧列表同步滚动并高亮下一项。
6. 用户重复步骤 1～5，直到本章 Flow 结束 → 提示进入下一 Act。

**要点**：一次「下一步」= Flow 中前进 **一个节点**，与游戏节奏 1:1 对齐。

---

### 流程 C：查漏补缺（上一步）

1. 用户想确认上一节是否漏做、或回忆 NPC/路线。
2. 用户点击右栏 **「上一步」**。
3. 系统行为：
   - 进度指针移到 Flow **上一节**；
   - 右栏加载上一节详情与配图；
   - 中栏地图与编号切换到上一节所在位置；
   - **默认保留** 更后节点的「已完成」勾选（用户可能只是查阅）。
4. 用户看完上一节后：
   - **若只是查漏**：点 **「返回当前进度」**（跳回指针曾到的最新推进位置），或点列表/地图上较后已完成的节点再点「设为我的当前节点」；
   - **若游戏进度确实还在上一节**：留在上一节，继续游戏后仍用「下一步」往后推。
5. 若用户从较早节点再次点「下一步」，系统按指针**逐节前进**；若与已有「已完成」冲突，可弹出一次确认：「是否取消之后节点的完成标记？」（产品可配置默认否）。

---

### 流程 D：用户点击认为对的当前节点（纠偏）

**场景**：游戏已做到 ⑤，网站指针还在 ③；或想先看清 ⑦ 的内容但还没做到。

#### D1 设为当前进度（纠偏指针）

1. 用户在地图点击 **⑤** 或在左侧列表点击第 5 项。
2. 右栏展示 ⑤ 的详情与配图（**正在查看 = ⑤**）。
3. 右栏出现 **「设为我的当前节点」**（或点击后二次确认）。
4. 用户确认后：
   - **进度指针**对齐到 ⑤；
   - ①～④ 可自动标为已完成，或保持原状（产品策略二选一，建议：④及之前标完成，与「游戏已过」一致）；
   - 「下一步提示」以 ⑤ 为当前；
   - 中栏 ⑤ 脉冲高亮。
5. 之后用户在游戏完成 ⑤，点「下一步」→ 进入 ⑥。

#### D2 仅查阅、不改变指针

1. 用户点击 **②**（已完成节）。
2. 右栏进入 **查阅模式**：展示 ② 详情与配图；顶部条「当前进度：⑤ · [返回当前进度]」。
3. 中栏高亮 ②，但不改变进度指针。
4. 用户点「返回当前进度」→ 右栏与中栏回到 ⑤。

#### D3 点击未来节点（剧透关）

1. 剧透关闭时，用户点击指针之后的 **⑧**。
2. 提示「尚未推进到此，仅可预览标题」或禁止展开配图/全文（按设置）。
3. 剧透开：允许查看，但不自动移动指针，除非用户点「设为我的当前节点」。

---

### 流程 E：地图编号点击与悬浮

#### E1 悬浮（未点击）

1. 鼠标移到 **③** 上 → 浮层显示：序号+标题、2～3 行要点、已完成/当前/未到。
2. 手机：**长按 0.5s** 同等浮层。

#### E2 点击

1. 点击 **③** → 进入流程 D（查阅或配合「设为当前」）。
2. 右栏刷新配图与说明；中栏 ③ 选中描边。

---

### 流程 F：左侧列表操作

1. 列表顺序 **与 Flow 完全一致**，序号与地图 ①②③ 一致。
2. 点击任一项 → 同流程 E/D。
3. 若该项不在当前截图 → 中栏**自动切换**到对应地图并滚动编号入视野。

---

### 流程 G：切换 Act（左侧）

1. 用户点击 **Act 2**。
2. 左栏 Act 2 选中；中栏加载 Act 2 默认地图；列表换为 Act 2 的 Flow。
3. 右栏显示 Act 2 的**进度指针所在节**（或该 Act 存档位置）。
4. 「下一步 / 上一步」仅在**当前 Act 的 Flow 内**有效。

---

### 流程 H：断点续玩

1. 用户关闭网站再打开。
2. 首页：「继续 Act 1 · 第 8/24 节 · xxx 节点名」。
3. 进入工作台：指针、地图编号、右栏详情、配图恢复到关闭前状态；中栏缩放比例可复位为「适应」（可选记忆上次缩放）。

---

### 流程 I：可选支线

1. 支线出现在列表「可选」分组，地图有「支」标。
2. 点击查看详情与配图；**「下一步」不改变主线指针**。
3. 用户可「标记支线完成」或「跳过」。

---

### 流程 L：中间地图缩放与平移（新增）

1. 中栏顶部工具条：**放大 [＋]**、**缩小 [－]**、**适应窗口**、**100% 复位**。
2. 用户点击放大：
   - 截图放大，编号点仍贴在对应地理位置（相对百分比不变）；
   - 若超出视窗，出现**拖拽平移**（按住拖动或触控滑动）。
3. 用户点击缩小 / 适应 / 复位：逐步缩小或整图适应中栏宽度。
4. 桌面：**Ctrl + 滚轮** 缩放（可选）；触屏：**双指捏合**。
5. 缩放仅影响**查看**，不改变进度指针；切换节点/切换地图时，可保留该图缩放状态（按 Act+mapId 记忆）或默认「适应」。

**边玩典型用法**：默认「适应」看全图 → 放大找传送点/门 → 对照右栏配图 → 完成后点「下一步」。

---

### 流程 M：右侧详情与配图（新增）

用户选中或推进到某节点后，右栏自上而下：

1. **进度摘要**：`Act 1 · 8/24 · 当前：xxx`
2. **下一步提示卡**（当前进度节点的行动摘要，最醒目）
3. **节点头图**（若有 `coverImage`）
4. **配图区**：节点级多图轮播/网格 + 每步附带的小图（有则显示在对应步骤下）
5. **文字说明**：步骤列表（阅读用，不强制逐步点完）
6. **操作区**：`[上一步]` `[下一步]` · 查阅模式下 `[返回当前进度]` `[设为我的当前节点]`
7. 用户**点击任意配图** → 全屏 Lightbox，可左右切换同节点图片，Esc 关闭

切换节点时，配图区**淡入切换**，避免误以为是同一张图。

## 五、三栏联动规则

```
进度指针 (flowIndex)
    ↔  左栏「当前」项 + 中栏脉冲编号 + 右栏「下一步提示」

正在查看 (selectedNode)
    ↔  右栏详情与配图 + 中栏选中描边 + 左栏高亮

「下一步」
    →  指针 +1，右栏与中栏跟 Flow 下一节

「上一步」
    →  指针 -1，右栏与中栏跟 Flow 上一节（查阅/回退）

用户点击其他节点
    →  仅改「正在查看」，直至「设为当前」或「返回当前进度」
```

**避免混淆**：查阅 ② 时，右栏必须显示「当前进度在 ⑤」的提示条，避免用户以为网站记录还在 ②。

## 六、移动端差异（简要）

1. 默认露出右栏「下一步提示」+ 操作按钮（底部固定）。
2. 地图通过「展开地图」进入全屏，支持双指缩放；点编号后底部抽屉显示详情与配图。
3. 配图点击全屏；「上一步 / 下一步」在拇指区。

---

## 七、验收走查（修订）

1. 新用户进入 → Flow 第 1 节 → 右栏有提示与配图 → 中栏 ① 高亮  
2. 游戏完成第 1 节 → 点「下一步」→ 自动到第 2 节，① 打勾，右栏配图切换  
3. 点「上一步」→ 回到第 1 节查阅 →「返回当前进度」回到第 2 节  
4. 游戏实际在第 5 节、网站在第 3 节 → 点 ⑤ →「设为我的当前节点」→ 指针到 5  
5. 中栏放大截图 → 编号仍对齐 → 复位适应  
6. 点击配图 → 全屏预览 → 关闭后回右栏  
7. 关闭重开 → 指针与查看节一致恢复  

---

## 八、与 PRD v0.1 的主要变化

| 原方案 | 现方案 |
|--------|--------|
| 步骤级「下一步」逐步解锁 | **节点级**「下一步」，与游戏过一节点同步 |
| 未解锁节点不可点 | 默认可点选查看；指针用 Flow +「设为当前」纠偏 |
| 右栏以文字为主 | 右栏**必须支持**头图、节点图、步骤图 |
| 地图静态示意 | 游戏截图 + **缩放/平移** |
| 前置 prerequisites 锁定 | **固定 flowOrder** 为主轴 |
