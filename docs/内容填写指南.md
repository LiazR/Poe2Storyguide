# 内容填写指南

所有剧情数据**只改 `content/` 文件夹**。

`npm run dev` 运行时会**自动**把 `content/` 同步到 `public/content/`。  
改完 JSON **保存 → 浏览器 F5 刷新** 即可，**不用**关进程再启动。

手动同步（可选）：`npm run content:sync`

---

## 一、文件在哪里

| 你要做的事 | 改哪个文件 / 放哪个文件夹 |
|------------|---------------------------|
| **添加 / 修改节点、坐标、名称** | `content/chapters/act1.json` |
| **添加新章节 Act 2** | 复制 `act1.json` → `act2.json`，并改 `content/manifest.json` |
| **放地图截图（PNG）** | `public/maps/` 文件夹 |
| **在 JSON 里引用地图** | `maps[].image` 填 `/maps/你的文件名.png` |

---

## 二、添加 PNG 地图截图

1. 把你的 PNG 放进：

   ```
   public/maps/act1-shore.png
   public/maps/act1-forest.png
   ```

2. 在 `content/chapters/act1.json` 的 `maps` 里对应填写：

   ```json
   {
     "id": "act1_shore",
     "title": "海难海岸",
     "image": "/maps/act1-shore.png"
   }
   ```

3. 运行 `npm run dev`（会自动 sync）；保存 JSON 后 **F5 刷新** 即可，无需重启进程。

> 若 PNG 还没放好，页面会暂时显示 SVG 占位图；放好 PNG 并 sync 后自动用你的图。

**多张地图**：在 `maps` 数组里多加一项，每个节点的 `mapId` 指向对应地图的 `id`。

---

## 三、修改节点位置（x / y）

坐标是 **0～100 的百分比**，相对地图左上角。

### 方法 A：调试模式（推荐）

1. 打开：http://localhost:5173/guide/act1?debug=1  
2. 按 **F12** → **Console（控制台）**  
3. **单击**地图（不要拖动）→ 控制台输出：`"x": 35.2, "y": 58.1`  
4. 地图上会出现：
   - **红圈** = 你刚点击的位置  
   - **绿圈** = JSON 里已有节点的坐标点  
   - **序号圆点中心** 应与绿圈重合；若不准，改 JSON 的 x/y  
5. 写入 `content/chapters/act1.json` → 保存 → F5 刷新  

> 坐标是相对于**图片**的百分比，不是整个浏览器窗口。debug 已修复为与节点使用同一参照（`.map-overlay`）。

### 在 F12 里查看节点位置

1. F12 → **Elements（元素）**  
2. `Ctrl+F` 搜索 `marker-wrap` 或节点 id（如 `data-node-id="node_intro"`）  
3. 选中后看：
   - **HTML 属性**：`data-x="22"` `data-y="68"`  
   - **Styles → element.style**：`left: 22%; top: 68%`  
4. 父元素应是 `div.map-overlay`（与图片同大小）

### 方法 B：肉眼微调

直接改 JSON 里的 `"x"`、`"y"`，每次改 1～2，sync 后看效果。

## 五、地图上显示的名称和彩色标签

每个节点在地图圆点**下方**会显示：

```
  ①          ← 序号圆点
  营地        ← mapTitle（没有则用 title）
  传送        ← badge.text（可选，建议 ≤6 字）
```

| 字段 | 作用 |
|------|------|
| `mapTitle` | 地图上显示的短名称（可选） |
| `title` | 未填 mapTitle 时用；也是左侧列表、右栏标题 |
| `badge.text` | 名称下方彩色小标签，最多约 6 字 |
| `badge.type` | 标签颜色类型（见下表） |

### badge.type 颜色对照

| type | 用途示例 | 颜色 |
|------|----------|------|
| `waypoint` | 传送点 | 蓝色 |
| `boss` | Boss | 红色 |
| `skill` | 技能点 / 神像 | 紫色 |
| `quest` | 主线 / 对话 | 金色 |
| `danger` | 危险区 | 橙色 |
| `loot` | 补给 / 宝箱 | 绿色 |

示例：

```json
"badge": { "text": "技能点", "type": "skill" }
```

不需要标签时，**删掉 `badge` 字段**即可。
